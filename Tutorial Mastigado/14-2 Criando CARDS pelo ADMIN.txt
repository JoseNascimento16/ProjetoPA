Vamos migrar a criação de cards (ex: novos planos de ação) a partir do Admin, ao invés de pelo código (Futuramente vamos migrar novamente para que possamos cria-los a partir do proprio site)

Como já temos tudo configurado no Admin, só precisamos alterar o código mesmo.

Para resumir o que vamos fazer: Ao criarmos um novo "plano de ação" no Admin, ele será guardado no banco de dados como um objeto a partir do modelo de "Planos_de_acao" que criamos em models.py(do app).
Vamos importar este modelo "Planos_de_acao" no views.py(do app), atribuir a uma variável, jogar as informações em um dicionário. E no HTML da página, iremos fazer uma varredura por este dicionário, para depois utilizar os dados desta varredura atravéz de código python no Html.

Em nome_do_app > views.py

	from .models import Plano_de_acao (classe criada como modelo)

	* a função ficará assim Ex:

	def planos_de_acao(request):
    	   planos = Plano_de_acao.objects.all()

           dados = {
             'chave_planos' : planos
           }

           return render(request, 'planos_de_acao.html', dados)

e no html teremos algo parecido com essa estrutura Ex:

{% if chave_planos %}
{% for elemento in chave_planos %}

<p class="clear-2 itens-cards-planos">
     <a href="#">
	{{ elemento.ano_referencia }}
     </a>
</p>

{% endfor %}
{% else %}
{% endif %}

** ano_referencia é um dos campos da classe modelo Plano_de_acao, ou seja, um valor específico que queremos mostrar no card.

Pronto, agora ao criarmos um novo objeto (plano de ação no nosso caso), ele será mostrado no site.
