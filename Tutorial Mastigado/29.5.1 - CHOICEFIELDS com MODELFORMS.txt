O choiceField é uma estrutura diferenciada, portanto, melhor exemplificar.

Depois de montarmos a nossa estrutura padrão de um modelform, ADICIONAMOS um modelo de FIELD manualmente e inserimos ele na call do MODELFORM:

## MODELO PADRAO ##

class FuncionariosForm(forms.ModelForm):


    class Meta:

        model = User
        fields = ['first_name', 'username', 'password']
        labels = {
		'first_name':'Nome do funcionário:',
           }
        widgets = {
		'first_name': forms.TextInput(attrs={'placeholder': 'Nome completo...','class': 'fonte-italic'}),
           }

## AGORA MODELO COM O CAMPO CHOICEFIELD ##

TIPO=[('-------','-------'),('Membro do colegiado','Membro do colegiado'),('Tesoureiro','Tesoureiro')]

# Um problema que ocorre, é que quando carregamos a página, o valor initial do CHOICEFIELD aparece aleatório. Podemos contornar isso ao setar esse valor INITIAL na INICIALIZAÇÃO do MODELFORM na VIEWS:

form_funcionarios = FuncionariosForm(initial={'funcao': '-------'})


## Este problema ocorre porque estamos usando o modelo padrão de "User" e ele não tem um choicesfield, são todos textfields
## Uma forma de contornar isso é criar um modelo separado só para o cargo e criar o field usando o models.choicesfield. Depois teríamos que criar uma foreingkey neste modelo associando ao User, ou seja, muito trabalho só para resolver este problema de ordenação de um choicefield.
## Um ultimo detalhe, perceba que na variável TIPO, usamos um dicionário {}, mas um dicionário é uma coleção do tipo SET, e um set é um tipo "não-ordenado". Para termos um set "ordenado" podemos usar listas [] ou tuplas (). No projeto final usei listas[]. PS: no exemplo acima ja troquei para []
