No urls.py do tutorial anterior, o video fala para nós deixarmos o path e o name='' conforme ele especificou, mas o nome dos nossos templates personalizados.html não precisa (já fiz esse teste).

No caso dos templates das urls 1, 2 e 4 do tutorial anterior, eu somente copiei o HTML atravéz do proprio navegador e fiz alguns ajustes simples.

Porém no template da url 3 que corresponde a url name='password_reset_confirm' não basta somente copiar o HTML pelo navegador.
O form que vem da cópia do navegador não nos traz o formato {{form}} que o backend precisa para processar o form.
Portanto, nós fizemos o seguinte:

	- Copiei o html do navegador e fiz um novo personalizado.html que cadastrei na url 3
	- Abri o template original (password_reset_confirm.html) no endereço especificado acima no inicio deste tutorial.
	- Copiei dele o trecho que carrega umas tags ( {% load i18n static %} )e colei dentro do <body> do nosso personalizado.html
	- Copiei dele TODA a estrutura do form e substitui pela estrutura do form que estava no nosso arquivo.html
	- Copiei dele também a estrutura do IF que verifica se o link de reset ainda é valido
	- e a estrutura do form no personalizado.html ficou assim:
########

{% if validlink %}
    
<p>Por favor, informe sua nova senha duas vezes para que possamos verificar se você a digitou corretamente.</p>

<form method="post">{% csrf_token %}
        <fieldset class="module aligned">
            <div class="form-row field-password1">
                {{ form.new_password1.errors }}
                <label for="id_new_password1">Nova senha:</label>
                {{ form.new_password1 }}
            </div>
            <div class="form-row field-password2">
                {{ form.new_password2.errors }}
                <label for="id_new_password2">Confirme a senha:</label>
                {{ form.new_password2 }}
            </div>
            <input type="submit" value="{% translate 'Change my password' %}">
        </fieldset>
        </form>

{% else %}

<p>{% translate "The password reset link was invalid, possibly because it has already been used.  Please request a new password reset." %}</p>
    
{% endif %}
