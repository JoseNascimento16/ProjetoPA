Agora precisamos fazer com que ao clicarmos no plano, sejamos direcionados para a 
pagina do plano corretamente.

Para isso precisamos fazer 2 tipos de alterações:
    1 - Precisamos indicar qual dos planos guardados no banco de dados será puxado.(plano 1, 2, 3...)
    2 - Precisamos alterar o html da pagina do plano para que ele mostre as informações
        vindas das variáveis, que são as informações no banco de dados.

1 - Neste passo vamos alterar código em varias partes do nosso projeto
    para informar o id do plano específico.

*Em qualquer link HTML que queiramos chamar um plano específico:

	<a href="{% url 'chamando_1_plano' elemento.id %}">

**Agora na urls.py (optei aqui pelo app de plano de acao)

	path('<int:elemento_id>',views.plano, name='chamando_1_plano')

***Agora na views.py a função ficou assim:
	
	def plano(request, elemento_id):
    		plano = get_object_or_404(Plano_de_acao, pk=elemento_id)

    		plano_a_exibir = {
        		'chave_planos' : plano
    		}
    		return render(request, 'plano.html', plano_a_exibir)

### Perceba que, desde o link, viemos trazendo as informações do "id" do elemento, que no nosso caso é o plano que foi utilizado para gerar aquele CARD onde clicamos no link para chamar a url "chamando_1_plano"
### Portanto, ao adicionarmos elemento.id , capturamos o id do plano que queremos.
### Em ** e *** temos o trecho "elemento_id" aparecendo 3 vezes. O trecho não precisa ser necessariamente "elemento_id", poderia ser qualquer coisa, contanto que nessas 3 aparições eles sejam iguais.

Pronto, agora já conseguimos acessar a página de plano, com um plano específico associado à ela.

2 - Como a página já foi renderizada recebendo o plano associado a ela como contexto, podemos embedar código python no html da página para mostrar as informações que quisermos deste plano.

Ex: 

<div class="box-plano1-1">
     <h5><i class="fas fa-chart-bar margem-icones-menu"></i>PLANO: {{ chave_planos.ano_referencia }}</h5>
</div>

Perceba que usamos o nome "chave_planos" (nome do dicionário utilizado para gerar o contexto) para identificar o elemento.
