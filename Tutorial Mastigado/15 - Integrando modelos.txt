O Django possui 3 formas de integrar modelos, usaremos aqui a forma "1 para varios", ou seja, 1 modelo PAI possui varios modelos FILHOS, porém cada modelo filho só possui 1 modelo PAI.

Como exemplo vamos utilizar planos de acao.

Já temos que ter no mínimo 2 modelos (usuarios e planos de acao) criados e importados para o banco de dados, e cadastrados no Admin

- Primeiramente temos que apagar todos os planos já criados lá pelo Admin mesmo.

- Agora vamos adicionar o vínculo de Usuarios em planos de acao.

Em planos_de_acao(app) > modelos.py

	Primeiro importamos a classe de usuarios:

        	from usuarios.models import Usuarios
		(ou seja, dos modelos do app usuarios, importe a classe Usuarios)

   	 E agora criamos a associação (dentro da classe Plano_de_acao)

       	 	usuario = models.ForeignKey(Usuarios, on_delete=models.CASCADE)

	ou seja, usuario será atribuido justamente como mais um elemento da classe "Plano_de_acao"

E pronto!! 

Exemplo: Queremos mostrar qual usuario esta associado a um determinado plano de ação.

- Ao criar um novo plano de ação agora, seja pelo Admin ou pelo site mesmo, temos que obrigatoriamente informar um usuario, pois agora, a classe Plano_de_acao tem um atributo "usuario" que tem que ser informado.

( Quando implementarmos o acesso autenticado de usuarios, essa informação ja vai poder ser atribuida automaticamente.)

- Portanto, ao criarmos um card (explicado no tutorial anterior) já podemos referenciar elementos da classe usuarios:

{% if chave_planos %}
        {% for elemento in chave_planos %}
            <div class="cards-planos">
                <p class="clear-2 itens-cards-planos">
                    <a href="{% url 'chamando_plano' %}">
                        <i class="fas fa-edit margem-icones-menu"></i>{{ elemento.ano_referencia }}
                    </a>
                <p class="clear-2 itens-cards-planos">
                    <a href="#">
                        <i class="fa fa-user margem-icones-menu"></i>{{ elemento.usuario.username }} <---- TIPO ASSIM ---------
                    </a>
                </p>
            </div>
        {% endfor %}
        {% else %}
        {% endif %}

Note que, em "elemento.ano_referencia", ano_referencia é um atributo da classe Plano_de_acao.
E em "elemento.usuario.username", username é um atributo de "usuario". E usuario passou a ser tambem um atributo de Plano_de_acao
